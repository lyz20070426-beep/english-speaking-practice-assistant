# 英语口语练习助手阶段性总结

更新时间：2026 年 5 月 16 日

## 一、项目定位

本项目目前定位为“大学生英语口语练习助手”的学校试用版原型。

第一阶段不追求复杂 App，也不优先做高成本平台，而是先围绕老师和学生最需要的功能：

- 学生能方便练单词、练口语。
- 系统能给出相对可靠的口语评分。
- 老师能看到学生练习数据。
- 后续可以低成本升级为云端正式版。

## 二、已经完成的主要工作

### 1. 项目资料和协作基础

- 建立 GitHub 仓库，用于项目同步和版本管理。
- 建立项目文档目录 `docs/`。
- 补充需求说明、开发计划、问题清单、协作同步方案等基础文档。
- 整理了试点单元词表，作为第一批练习素材。

### 2. 学生端练习功能

已完成学生端核心页面和交互：

- 单词列表。
- 单词详情。
- 音标、中文释义、例句展示。
- 标准发音播放。
- 录音跟读。
- 录音回放。
- 错词/需复习标记。
- 练习记录展示。
- CSV 记录导出。

### 3. 讯飞正式评分

已接入讯飞语音评测能力，解决浏览器原生语音识别不稳定的问题。

当前能力：

- 浏览器录音。
- 音频转为 16k 单声道 PCM。
- 后端调用讯飞语音评测接口。
- 返回总分、准确度、流利度、完整度、音素分、单词分等结果。
- 页面展示讯飞评分。
- 保存评分记录。

说明：

- 讯飞密钥已放在本地后端 `.env` 中，不进入前端页面。
- GitHub Pages 静态页面不能直接调用讯飞正式评分，正式评分需要后端服务。

### 4. 麦克风检测与设备选择

针对测试中出现的浏览器或设备录不到声音的问题，已增加产品内自检能力：

- 页面中可以选择麦克风设备。
- 可以点击“检测麦克风”。
- 页面显示麦克风输入音量。
- 如果没有检测到有效声音，会提示用户切换设备或浏览器。

这一步降低了老师和学生使用时遇到录音问题的排查难度。

### 5. 学生身份信息

学生端已从单一“学生姓名”升级为完整身份信息：

- 学生姓名。
- 手机号。
- 学号，可选。
- 班级，可选。

这些字段会写入练习记录，并在老师后台和导出表格中使用。

当前仍属于免短信的简化版，后续可以平滑升级为短信验证码登录。

### 6. 系统点评和学习建议

系统不只显示分数，还会根据评分自动生成点评。

示例：

- 优秀：读音清楚，整体表现稳定，可继续挑战句子朗读。
- 良好：基本会读，建议再跟读 1-2 遍，重点保持重音和结尾音。
- 需加强：发音准确度、朗读流利度偏弱，建议先听标准发音，再慢速跟读。
- 重点复习：建议先确认麦克风录音清楚，再听标准发音重新练习。

点评会出现在：

- 学生练习记录。
- 老师后台学生概览。
- 导出的 CSV 表格。

### 7. 情景对话

已完成基础情景对话练习：

- 默认 4 个情景。
- 学生可以选择场景。
- 系统给出开场问题和追问反馈。

目前仍是模板式情景对话，暂未接入真正 AI 对话。

### 8. 老师后台雏形

已新增“老师后台”页面。

当前可以查看：

- 学生人数。
- 练习次数。
- 班级均分。
- 薄弱单词数量。
- 学生练习概览。
- 学生平均分。
- 最近练习时间。
- 需复习单词。
- 系统总体点评。
- 班级薄弱单词榜。

当前老师后台读取的是本地浏览器中的练习数据。后续接入云数据库后，可升级为全班真实数据后台。

### 9. 老师端内容管理

已在老师后台增加本地内容管理能力。

#### 课程单词导入

老师可以粘贴单词表，格式为：

```text
单词, 音标, 中文释义, 例句, 是否重点
```

导入后，学生端单词列表会立即更新。

#### 情景对话导入

老师可以粘贴情景对话，格式为：

```text
场景名称 | 开场问题 | 追问1 | 追问2 | 结束建议
```

导入后，学生端情景对话列表会立即更新。

## 四、当前版本的使用方式

### 本地正式评分版本

需要先启动后端：

```powershell
& "C:\Users\MSI\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe" server/server.js
```

然后浏览器打开：

```text
http://127.0.0.1:8787/
```

### GitHub Pages 静态展示版本

可访问：

```text
https://lyz20070426-beep.github.io/english-speaking-practice-assistant/app/
```

说明：GitHub Pages 版本可以展示前端页面，但讯飞正式评分需要后端支持，不能只靠 GitHub Pages 完成。

## 五、当前版本的边界

当前版本已经适合进行小范围演示和流程验证，但还不是正式全班使用版本。

主要边界：

- 数据仍主要保存在本地浏览器。
- 老师后台暂时不能看到不同学生手机上的统一数据。
- 学生登录还不是短信验证码登录。
- 讯飞评分后端仍在本地电脑运行。
- 情景对话还是模板式，不是真正 AI 实时对话。
- 录音文件暂未做云端长期保存。

## 六、下一步建议

### 优先级 1：给老师试用和收集反馈

建议先让李老师看当前版本，重点听取以下反馈：

- 单词跟读和讯飞评分是否符合教学需要。
- 老师后台统计指标是否够用。
- 是否需要保存录音供老师回听。
- 单词导入格式是否方便老师使用。
- 情景对话管理是否符合课堂内容。

### 优先级 2：准备云端试用版

如果老师认可方向，下一步建议进入云端试用版开发：

- 购买或准备轻量云服务器。
- 部署后端服务。
- 接入数据库。
- 将本地练习记录改为云端保存。
- 老师后台读取全班真实数据。

### 优先级 3：登录和权限

云端版稳定后，再做：

- 学生手机号登录。
- 老师账号登录。
- 班级权限控制。
- 后续再考虑短信验证码。

### 优先级 4：更强的教学功能

后续可继续扩展：

- 按单元布置练习任务。
- 学生完成率统计。
- 老师查看某个学生完整练习轨迹。
- 句子朗读评分。
- 课文段落朗读评分。
- 真正 AI 情景对话。
- 周报或学习报告。

## 七、阶段结论

目前项目已经从最初的“单词朗读 demo”推进到“具备学生端、老师后台、正式评分、内容导入、系统点评和正式版方案”的试用版原型。

下一阶段最重要的不是继续堆功能，而是让老师实际看一遍，确认教学场景是否匹配。确认后，再进入云端多人试用版开发。
