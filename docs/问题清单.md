# 问题清单

## 一、需求问题

- [ ] 第一版主要给谁使用：女儿个人、一个班级，还是整个学校？
- [ ] 学校是否已经明确要求必须有老师端？
- [ ] 老师最关心的是学生是否完成，还是发音质量？
- [ ] 学生练习内容来自课本、单词表、老师上传内容，还是系统内置内容？
- [ ] 第一版是否需要支持多个班级？
- [ ] 是否需要家长端？

## 二、账号和权限

- [ ] 学生是否需要账号密码登录？
- [ ] 是否可以先用学生姓名或学号进入？
- [ ] 老师账号由谁创建？
- [ ] 管理员是否需要单独后台？
- [ ] 女儿在学校使用时，电脑环境是否允许登录外部网站？

## 三、口语练习方式

- [ ] 第一版是先支持单词，还是句子和对话也一起做？
- [ ] 是否必须有系统示范音频？
- [ ] 示范音频来源是什么：人工录制、浏览器语音合成、AI 生成，还是第三方词典？
- [ ] 学生录音是否需要长期保存？
- [ ] 是否允许老师听学生录音？

## 四、口语评分

- [ ] 第一版是否必须自动评分？
- [ ] 如果要自动评分，评分维度包括哪些：准确度、流利度、完整度、发音？
- [ ] 是否接受先做录音回放和完成记录，后续再增加评分？
- [ ] 是否考虑使用第三方语音评测服务？
- [ ] 是否有费用预算？

## 五、数据和隐私

- [ ] 学生录音属于个人数据，学校是否有隐私要求？
- [ ] 数据需要保存在哪里：本地服务器、云服务器，还是学校服务器？
- [ ] 录音保存多久？
- [ ] 是否需要删除学生历史数据的功能？

## 六、部署和使用环境

- [ ] 系统是在学校局域网使用，还是互联网访问？
- [ ] 学生主要使用电脑、平板还是手机？
- [ ] 学校电脑是否有麦克风？
- [ ] 学校浏览器是否支持录音权限？
- [ ] 是否需要支持手机微信内打开？

## 七、素材整理

- [ ] 当前 `单词表` 的文件格式是什么？
- [ ] 单词表是否包含中文释义、音标、例句？
- [ ] 是否需要按年级、单元、课本版本分类？
- [ ] 老师聊天截图中是否有明确的练习要求？
- [ ] 是否有现成课文或听力材料？

## 八、协作问题

- [ ] 是否已经创建 GitHub 私有仓库？
- [ ] 女儿是否有 GitHub 账号？
- [ ] 女儿学校电脑是否可以访问 GitHub？
- [ ] 是否需要准备一个共享文档收集老师反馈？
- [ ] 是否需要每周固定整理一次项目文档？

## 九、下一步建议先确认

建议优先确认以下 5 个问题：

1. 第一版是给个人试用、班级试用，还是学校试用？
2. 第一版是否必须自动评分？
3. 学生录音是否需要保存并让老师查看？
4. 练习素材优先从哪个单词表或课本开始？
5. 系统将来准备部署在哪里？
