# 部署与费用估算

## 一、部署目标

正式试用版要解决三个问题：

- 武汉学生可以通过正式网址访问学生端。
- 老师可以通过后台查看全班练习数据。
- 讯飞密钥安全保存在服务器端，不暴露给浏览器。

## 二、推荐部署方案

### 方案 A：继续本地演示

适合阶段：家里测试、老师现场演示。

组成：

- 前端：本地或 GitHub Pages。
- 后端：本机运行 `server/server.js`。
- 数据：浏览器本地存储。
- 讯飞：使用已开通的个人免费包。

费用：0 元。

缺点：

- 女儿和老师在外地不能直接共用同一套数据。
- 老师后台只能看当前电脑或当前浏览器数据。
- 不适合全班真实试用。

### 方案 B：轻量云服务器

适合阶段：学校小规模试用。

组成：

- 前端：部署到服务器或静态托管。
- 后端：Node.js 服务。
- 数据库：SQLite 起步，后续可换 MySQL/PostgreSQL。
- HTTPS：免费证书。
- 讯飞：后端统一调用。

优点：

- 架构简单。
- 费用低。
- 适合当前项目阶段。
- 后端和数据库都在一台服务器上，维护相对直观。

建议优先采用此方案。

### 方案 C：云数据库 + 独立后端

适合阶段：试用人数增加后。

组成：

- 前端静态托管。
- 后端服务。
- 云数据库。
- 对象存储保存录音。
- 短信服务。

优点：

- 更规范。
- 扩展性更好。

缺点：

- 成本更高。
- 配置更复杂。
- 当前阶段暂不必马上做。

## 三、费用估算

费用会随云厂商活动、地区、配置和实际调用量变化，以下为项目规划估算，正式采购前应以控制台价格为准。

### 1. 云服务器

学校试用版可先选择轻量应用服务器。

预估：

- 活动价：几十元到一两百元/年可能出现。
- 常规低配：约几十元/月。
- 第一阶段建议预算：100-300 元/年。

建议配置：

- 2 核 CPU。
- 2GB 内存。
- 40GB 以上磁盘。
- 适合小规模试用。

### 2. 域名

如果需要正式网址，建议购买域名。

预估：

- 约 50-100 元/年。

也可以先不用独立域名，直接用云服务器提供的访问地址测试。

### 3. HTTPS 证书

可使用免费证书。

预估：

- 0 元。

### 4. 讯飞语音评测

当前已经开通个人免费套餐，可先用于试用。

预估：

- 试用阶段：优先使用免费包。
- 正式稳定使用：根据调用量购买套餐。
- 预算建议：先按 0 元试用，后续预留 1000 元/年以内的可能空间。

如果学生数量和练习次数增加，讯飞会成为主要变动成本。

### 5. 短信验证码

第一阶段不建议接短信验证码。

原因：

- 需要签名和模板审核。
- 需要按条计费。
- 会增加开发复杂度。

替代方案：

- 先用手机号作为身份字段。
- 老师发放班级码或登录码。
- 后续真实试点稳定后再接短信。

预估：

- 暂时 0 元。
- 后续按短信条数计费。

### 6. 数据库和录音存储

第一阶段：

- SQLite：0 元。
- 不长期保存录音：0 元。

后续如果保存录音：

- 需要对象存储。
- 费用取决于录音数量、大小和保存时长。

## 四、推荐预算

### 最小云端试用版

适合 1 个班或少量学生试用。

预算：

- 云服务器：100-300 元/年。
- 域名：0-100 元/年。
- HTTPS：0 元。
- 讯飞：先用免费额度。
- 短信：0 元。

合计：约 100-400 元/年。

### 稳妥试用版

适合老师正式组织学生试用一段时间。

预算：

- 云服务器：300-600 元/年。
- 域名：50-100 元/年。
- 讯飞套餐或备用费用：0-1000 元/年。
- 短信验证码：暂不接入，后续另算。

合计：约 350-1700 元/年。

## 五、下一步实施建议

建议按以下顺序推进：

1. 先保留当前本地试用版继续给老师看。
2. 确认老师是否认可学生端和老师后台方向。
3. 如果老师认可，再购买轻量云服务器。
4. 将当前后端部署到云服务器。
5. 接入数据库。
6. 把本地练习记录改为云端保存。
7. 最后再考虑短信验证码和录音长期保存。

## 六、费用依据说明

- 阿里云短信服务官方说明：短信支持按量计费和套餐包，国内短信需要签名和模板审核。
- 阿里云国内短信价格在 2026 年有调整公告，正式费用应以控制台实际价格为准。
- 讯飞语音评测官方页面说明其能力覆盖中英文词、句、篇评分，具体套餐价格需登录开放平台确认。
- 云服务器价格受活动、地域、规格影响明显，正式购买前需要在阿里云或腾讯云控制台确认。
