# 讯飞语音评测接入说明

## 当前目标

把浏览器不稳定的免费 `SpeechRecognition` 评分，升级为正式的讯飞语音评测评分。

## 本地启动步骤

### 1. 创建本地密钥文件

复制：

```text
server/.env.example
```

为：

```text
server/.env
```

然后填入讯飞控制台里的三个参数：

```text
XF_APPID=你的APPID
XF_API_KEY=你的APIKey
XF_API_SECRET=你的APISecret
PORT=8787
```

注意：`server/.env` 已加入 `.gitignore`，不要提交到 GitHub。

### 2. 启动本地后端

在项目根目录执行：

```bash
"C:\Users\MSI\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe" server/server.js
```

启动成功后会看到：

```text
英语口语练习助手后端已启动：http://127.0.0.1:8787
```

### 3. 打开本地页面

浏览器打开：

```text
http://127.0.0.1:8787/
```

使用“讯飞正式评分”按钮测试。

## 为什么不能直接在 GitHub Pages 上调用讯飞

讯飞接口需要 `APIKey` 和 `APISecret` 签名。如果把密钥写进前端页面，任何人都能在浏览器里看到并盗用。

因此正式评分必须通过后端：

```text
浏览器录音 -> 本地/服务器后端 -> 讯飞语音评测 -> 后端返回评分 -> 页面展示
```

## 当前实现

- 前端录制约 3 秒音频。
- 前端把音频转换为 16k 单声道 PCM。
- 后端读取 `.env` 中的讯飞密钥。
- 后端调用讯飞语音评测流式 WebSocket 接口。
- 后端解析评分 XML，返回总分、准确度、流利度等结果。

## 后续部署

如果要让女儿、老师在外地也能使用“讯飞正式评分”，需要把 `server/server.js` 部署到一个 HTTPS 后端服务器，例如：

- 云服务器
- Render
- Railway
- 阿里云/腾讯云轻量服务器
- Cloudflare Workers 方案需要另行适配 WebSocket 客户端和密钥存储

GitHub Pages 只能托管静态网页，不能安全保存讯飞密钥，也不能单独承担正式评分后端。
