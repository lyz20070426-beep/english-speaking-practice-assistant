# 需求说明

## 一、项目背景

学校希望开发一个英语口语练习助手，帮助学生在课堂外进行更高频、更自主的英语口语练习。学生可以根据老师布置的单词、句子、课文或考试题型进行朗读、跟读和对话练习。

爸爸负责整体规划、技术推进和项目资料整理，女儿在武汉学校提供真实使用场景、学生体验反馈和学校侧需求信息。

## 二、目标用户

### 学生

- 需要反复练习英语发音、单词、句子和对话。
- 希望获得及时反馈，而不是只能等老师批改。
- 需要一个简单、容易上手、没有压力的练习环境。

### 老师

- 希望布置口语练习任务。
- 希望查看学生是否完成练习。
- 希望了解学生常见发音问题和练习薄弱点。

### 家长

- 希望了解孩子是否练习、练习了什么、练习效果如何。
- 希望系统能辅助孩子形成英语开口习惯。

## 三、核心使用场景

### 场景 1：学生练习单词发音

学生选择一个单词表，系统显示英文单词、中文含义和示范发音。学生点击录音后朗读单词，系统给出发音反馈。

### 场景 2：学生练习句子朗读

学生按课文或老师布置内容朗读句子。系统对流利度、完整度和发音进行基础评价。

### 场景 3：学生进行对话练习

系统扮演对话中的一个角色，学生扮演另一个角色，进行简单英语问答或情景对话。

### 场景 4：老师布置练习任务

老师选择练习内容，设置班级、截止时间和练习要求。学生登录后看到自己的任务。

### 场景 5：查看练习记录

学生、老师或家长可以查看练习次数、完成情况、历史录音和系统反馈。

## 四、第一版功能范围

第一版建议只做最小可用版本，优先保证核心体验跑通。

详细产品边界见 [产品定义](产品定义.md)。

### 学生端

- 查看第一个试点单元的单词。
- 播放浏览器标准发音。
- 录制自己的朗读并回放。
- 标记“会读了”或“需复习”。
- 进入模板式情景对话。
- 查看历史练习记录。

### 老师端

- 1.0 版先不做复杂老师后台。
- 可通过演示页面查看整体功能效果。
- 后续再扩展任务布置和学生完成情况。

### 管理端

- 管理学生、班级、老师账号。
- 管理练习素材。
- 查看系统基础数据。

## 五、后续可扩展功能

- AI 对话练习。
- 发音评分和具体音素纠错。
- 口语考试模拟。
- 班级排行榜或激励体系。
- 微信或企业微信通知。
- 老师批注学生录音。
- 家长查看周报。

## 六、非功能需求

- 操作要简单，学生不需要复杂学习成本。
- 页面要适合学校电脑和普通家用电脑使用。
- 数据要能区分学生、班级和练习任务。
- 学生录音和个人数据需要注意隐私保护。
- 系统要方便后续增加更多练习素材。
